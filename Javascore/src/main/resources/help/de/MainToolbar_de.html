JavaScore - Arbeitsbereich und Menüs

(Tip 1: Es ist übersichtlicher, wenn der Javascore Arbeitsbereich auf den gesamten Bildschirm maximiert ist. Tip 2: Die Dialogfelder können mit der Maus größer gezogen werden, wenn Text oder Eingabefelder nicht vollständig dargestellt werden.)

Regattadaten Bearbeiten Schaltfläche - Ruft das Dialogfeld zur Erstellung oder Bearbeitung der allgemeinen Regattadaten auf.  Dieses Feld kommt automatisch, wenn noch keine Regatta geöffnet war.

Teilnehmer Eingeben/Bearbeiten Schaltfläche - Ruft das Dialogfeld zur Eingabe oder Bearbeitung der Teilnehmerdaten auf.

Wettfahrten Eingeben/Bearbeiten Schaltfläche - Ruft das Dialogfeld zur Eingabe oder Bearbeitung der Wettfahrten auf.

Datei Menü - Alles was mit Speichern und Aufrufen von Regatta Dateien, sowie Import und Export zu tun hat.

  • Regatta neu - Erstellt eine neue Regatta und ruft das "Regatta Bearbeiten" Dialogfeld auf.

  • Regatta öffnen -Öffnet eine schon gespeicherte Regatta von Diskette, Festplatte o.a.

  • Regatta sichern - Speichert die aktuelle Regatta

  • Regatta sichern als ... -Speichert die aktuelle Regatta unter einem wählbaren (anderen) Namen.

  • Regatta wiederherstellen - Stellt eine Regattadatei aus einer Backupdatei wieder her, die aktuelle Datei wird verworfen. Hintergrund: Von jeder Regattadatei wird beim Öffnen oder Speichern automatisch ein Backup im Javascore Verzeichnis erstellt. Man erkennt sie an der nachgestellten Tilde (Test.regatta~).

  • Import - Stellt eine Importmöglichkeit (für Teilnehmer, Ergebnisse, Rundungen) zur Verfügung von Excel oder Starcalc über die Zwischenablage nach Javascore.  Hier gibt es mehr Einzelheiten.

  • Export -  Stellt eine Exportmöglichkeit von Regattadateien in Textdateien (.txt,.csv, .tab, .asc, .prn) zur Verfügung.

  • Einstellungen - Ermöglicht allgemeine Programmeinstellungen zu ändern. Hier gibt es mehr Einzelheiten.

  • Schliessen - Schließt JavaScore

Einiges zum Speichern von Regatten. Die folgenden Funktionsabläufe im Programm sollen Informationsverlust vermeiden helfen.

  • Wenn eine Regatta erstellt wird, bekommt sie einen vorrübergehenden Namen bis zur ersten wirklichen Speicherung.

  • Wenn eine Regatta geöffnet oder erstellt wird, wird gleichzeitig eine Backupdatei mit der Dateiendung '*.regatta~' erzeugt. Der Inhalt dieser Backupdatei wird nicht angerührt bis die Regatta explizit gespeichert oder wiedergeöffnet wird.

  • Jedesmal wenn man die Dialogfelder 'Regatta Bearbeiten, Teilnehmer Eingeben/Bearbeiten, Wettfahrten Eingeben/Bearbeiten' schließt, speichert Javascore die Regatta; die Backupdatei wird dann nicht aktualisiert.

  • Wenn eine Regatta mit 'Speichern als...' gespeichert wird, wird die alte zugehörige Backupdatei gelöscht und eine neue von der gerade gespeicherten erstellt.

  • Wenn eine vorhergehend gespeicherte Regatta geöffnet wird, wird (wenn vorhanden) die zugehörige Backupdatei gelöscht und eine neue erstellt.

  • Wenn eine Regatta wiederhergestellt wird, dann wird die aktuelle Datei verworfen und die letzte '.regatta~' wird geladen.

  • Berichte werden jedesmal neu generiert, wenn eine Regatta gespeichert wird, ob das automatisch durch Javascore passiert, durch den Anwender erfolgt oder ob man den Bericht wieder neu aufruft. Das mag als Overkill gesehen werden, verhindert aber ziemlich zuverlässig, das die Regattadatei und die Berichte nicht mehr synchron sind.

Berichte Menü -  HTML Sofort-Ausgabe und Drucken aller Listen, Übersichten und Ergebnisse.

  • Berichte anzeigen - Ruft einen Browser zur Anzeige der Berichte auf. Das Program versucht immer den eingestellten Standard Browser zu öffnen. Wenn das nicht geht oder es soll ein anderer Browser benutzt werden, so kann das unter 'Einstellungen' im 'Datei' Menü umgestellt werden. Zum Drucken der Berichte kann ganz einfach die Druckfunktion des Browsers genommen werden.

  • Berichte Einstellungen - Ermöglicht die Einstellung der verschiedenen Optionen für das Ausgabeformat der Berichte. Hier gibt es mehr Einzelheiten.

Klassen/Flotten Menü - Hier werden alle Grund-Einstellungen zu den Einteilungen in , Gruppen und Teilgruppen einer Regatta gemacht.

  • Klassen/Flotten -Normalerweise wird hier, wenn es nur eine Segelklasse oder Flotte gibt, alles eingestellt.

  • Gruppen - Mehrere Klassen oder Flotten können hier zu einer Gruppe zusammengefaßt werden

  • Teilgruppen - Hier können Teilgruppen einer oder mehrerer Klassen/Flotten bestimmt werden. (z.B. Junioren, Frauen etc.)

Hilfe Menü - ruft die Programmhilfe und ein Programminfo auf.
