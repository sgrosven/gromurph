JavaScore Main Toolbar and Menu

Regatta Button - Brings up the regatta panel to allow you edit the overall regatta fields.  If no regatta has been opened, the Open Regatta dialog automatically comes up.

Entries Button - Brings up the entries panel to allow you to enter and edit the boats entered in the regatta (As of release 0.01, all boats must be one-design and are in the same division)

Races Button - Brings up the races panel to allow you to define or edit races, and enter finishes

File Menu - items related to saving and reading files from disk

  • New Regatta - creates a new regatta and brings up the regatta panel
  • Save Regatta - saves the current regatta to disk
  • Save Regatta As - saves the current regatta to disk allowing you to set a new file name
  • Import Regatta - provides an import capability from Excel via the "clipboard".  Click here for more details.
  • Restore Regatta - when you open a regatta, or explicitly save it, I create an "backup" version called .regatta~ (you may have seen these cluttering up your directory.  If you ask to Restore, I'll discard the current regatta entirely and reload from the backup.
  • Preferences Panel - Allows you to modify the overall program preferences. Click here for more details
  • Exit - exits JavaScore

Speaking of saving regattas, the following scheme is used to keep you from losing your information:

  • when you create a regatta, I give it a temporary file name until you do your first explicit save
  • when a regatta is opened or created, a "backup" of the regatta with the extensions '.regatta~' is created.  The contents of this backup are not touched until you either: explicitly save the regatta, or you open the regatta again JavaScore .
  • Whenever you exit out of the Regatta form, the Entries form, or the Races form, I automatically save the regatta (under the .regatta name, the backup is left untouched).
  • When you explicitly Save the regatta, I delete the old backup version and create a new one with the just saved regatta.
  • When you open a previously saved regatta, I delete the old backup file (if any) and create a new one.
  • If you RESTORE the regatta, I dump the current regatta and reload the latest "regatta~" regatta.
  • Reports are re-generated whenever the regatta is saved, whether JavaScore does in automatically, or you explicitly save the regatta, or you go to view the regatta's reports.   This is probably overkill, but it should make it very difficult for the reports to get out of synch with the regatta file (unless you have manually renamed a backup 'regatta~'  but since re-opened the regatta and given JavaScore a chance to update the reports)

Reports Menu - 

  • Show Reports - Brings a browser to display reports.  The program will, by default, try to use the default browser you have set on your computer.  If this doesn't work, or you want to use something different, use the Preferences panel under the  File menu to change it.  Use the browser's capabilities to print reports.
  • Report Options - Allows you to set the various options that control formatting of the reports. For more details, click here.

Master Files Menu - not yet implemented, will provide access to master database of boats and of divisions

Help menu - provides the ability to bring up the help system, and the about menu
