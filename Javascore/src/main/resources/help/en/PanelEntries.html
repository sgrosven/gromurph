JavaScore - Entries Screen

This panel edits information about entries.   The window has two main components.  On the left is a "tree" of all entries by class.  On the right is the details about the selected entry (if any).   

To open a class in the tree and show its entries, click on the toggle () so that it looks like this ().  When you click on an entry in the tree, its details will be show on the right.

Press Add to add a new entry (select a class first, and that class will be the new entry's initial division).

Select one or more entries, and press Delete to delete entries.

In the Entry Panel itself, the following data is currently maintained about each entry.  All of this information is optional, but I strongly suggest that you give each boat at least a  sail number, and either a boat name or a skipper's name.  The rest is up to you.  

Class:  this choice box tells the class in which the boat is entered.  You set the classes in the event on the main regatta screen.  By default, the selected class for a new entry is the same class as the previous entry.  If you have only one class, leaving class of "<none>" is OK (but ugly, I wouldn't do it if I were you, but I recognize that often single class imports may not have a class assigned).  If you have more than one class in the event, boats without a class assignment will not be scored.

Rating: This only shows when the class is a handicap class, enter the boat's rating

Sail#: the sail number of the boat.  This can be anything you'd like.  I will parse into into 3 pieces: an initial alphabetic portion, a center number, and a trailing alphabetic portion.  None of the three are required, but they correspond to the typical sailing number:  Country Number {trailer}.  The 'trailer' is not often used, but countries like England sometimes have numbers like:  "GBR 2345 Y".
    Duplicates are accepted, although I will warn you if you enter a duplicate.

Bow#: the bow number of the boat.  This field only appears if the "Bow numbers" parameter is selected on the main regatta page.  Like the Sail#, this does not have to be a number, I accept any string.  I also will accept duplicates, but will warn you first.

A warning will be generated for Bow numbers that match Sail numbers and vica versa.

Name:  The boat's name (If there is one, blank is fine).

Team Table: In each row, enter the last name, first name and the isaf id (if any) of the skipper and crew. If it is a syndicate name, I recommend putting the entire name as the last name rather than part of it as the first name. You can enter as many crew as you want, just keep putting new crew names in the "*" row and a new row for the next crew member will be appear.

If you enter a sailor's ISAF sailor id, then I'll automatically create a link around the sailor's name in all the reports to point to their biographical information on the ISAF web site (www.sailing.org).  For more information about this free service from ISAF, see: http://www.sailing.org/isafsailor/.

"CBYRA#" or "Rsa" or something else:  This is the field for the "Regional Sailing Association membership number.  I'm vague about the specific text you might see, but the end-user can set the label for this field.  See below.  By default, I ship with it set to "CBYRA" (for Chesapeake Bay Yacht Racing Association), that's only because that's where the author is from.   You can change this label in the Preferences panel.

"US Sailing#" or "Mna" or something else.  This is where the entry's membership number for their Member National Authority should be entered.  Likewise, the user can set the label for this, I ship with it set to "US Sailing". You can change this label in the Preferences panel.

Club: This should be the home club for the entry

SubDivisions: This panel only appears when you have defined one or more sub-divisions within the regatta.  If it appears, then check each sub-division in which the entry ought to be a part.
