Java Score - Import Dialog

There are now three separate dialogs for importing information: importing entry information, importing finishes, and importing mark roundings.  All three dialogs work about the same.  This file uses the importing entries screen as an example.

This dialog lets you import regatta entries and finishes from an Excel spreadsheet.  The spreadsheet in Excel show be arranged with each row being an entry and the columns being the information about the entry (skipper name, boat name, sail id, finishes if any, etc).  here are the steps:

(1) In Excel, select the rows you want to import to JavaScore.  "Copy" this block of cells to your clipboard ( on the menu bar, Edit, Copy):

(2) Next go to the Import Dialog in JavaScore by selecting the menu item "File", "Import Regatta...".  In that dialog click on the Paste button, and JavaScore will create a table of the data it finds on the clip board.

(3) The screen will show you a left hand column full of "choice boxes" that lets you tell JavaScore what each field in the import should become in JavaScore.   The next 3 columns are the first 3 rows of your import data set. The following screen shot shows the pasted cells from Excel, with the first three variable names already set, and the "Bow" field about to be registered.

(4) Press the OK key, a new Regatta is created at the data from this table is used to populate the regatta.  

If there are any race results contained in the selected variable,  new races will be create as necessary and populated with finishes.

The only time Finish Points are saved is when it matches a RDR or MAN penalty.  Otherwise the points will be overwritten when JavaScore turns around and does its own calculations.

Note also, that if you import FinishPosition, that should be the position not necessarily the points assigned, and the penalty must be entered via a separate field.

This is not terribly bullet proof yet.  I strongly suggest that you ALWAYS include the Sail Id and Helmsman Last Name (at least) in the import.  

I'm not very graceful yet at handling problems, and I will flag every error separately, so if you have a problem in the same cell in every row of your import, you may get the same error for each row.  Yes its ugly, but fixing that is a lower priority than other things right now. 

(5) If you change your mind and don't want to do the import, press the Cancel button

Side Notes for Techies and other computer savvy users:  The format on the clipboard is a very simple one, the fields are text-based, one row at a time, the cells in a row are separated by the "Tab" character, and the rows are separated by the "carriage return".  This is the "Unformatted Text" format that Excel puts on a clipboard.  I have not checked but this format could probably easily be replicated by a variety of means.  (i.e. you are probably not limited to importing from Excel, any tab/return based file should be "pastable")
